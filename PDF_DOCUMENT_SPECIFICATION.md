# PDF Document Structure Specification

This document defines the visual and functional structure for PDF documents generated or viewed within the Adoption & Child Care system.

## 1. Document Content Structure (The Template)

Every official PDF report (Child Profile, Medical Report, Case Summary) should follow this standardized layout:

### A. Header (Uniform across all reports)
- **Top Left:** Official System Logo (Adoption & Child Care Tracking System).
- **Top Right:** 
    - Report Type (e.g., "CONFIDENTIAL CASE REPORT")
    - Date of Issue: YYYY-MM-DD
    - Document ID: [Auto-generated Unique ID]
- **Center:** "PRIVATE & CONFIDENTIAL" (In red, bold).

### B. Identifying Information Section
A shaded box containing:
- **Child Details:** Name, Case Number, Date of Birth.
- **Worker Details:** Assigned Case Worker, Department/County.
- **Guardian/Family:** Primary Contact Name (if applicable).

### C. Main Content Area
- **Dynamic Sections:** Varies by report type (e.g., Medical History, Placement Logs, Legal Status).
- **Formatting:** 
    - Headers: Bold, 12pt, Dark Blue.
    - Body Text: 10pt, Sans-serif (Arial/Roboto).
    - Tables: Clean borders, alternating row colors for readability.

### D. Footer (Uniform across all reports)
- **Bottom Left:** "Generated by Adoption & Child Care Tracking System".
- **Bottom Center:** Page [x] of [y].
- **Bottom Right:** System Timestamp & Digital Signature/QR Code for verification.

## 2. In-App Viewer Structure (UX/UI)

When a user opens a PDF *inside* the Android app, the UI should provide:

### Top Navigation Bar
- **Back Button:** Return to Document List.
- **Title:** The file name (e.g., `Sam_Karanja_Medical_Report.pdf`).
- **Action Icons:** 
    - **Download:** Save to local storage.
    - **Share:** Open system share sheet.
    - **Print:** Send to connected printer.

### PDF Content View
- **Smooth Scrolling:** Vertical scrolling for multi-page documents.
- **Pinch-to-Zoom:** Allow users to zoom in on small text or signatures.
- **Page Indicator:** Floating bubble (e.g., "3/10") that appears during scroll.

### Interactivity (Before/After Download)
- **Before Download (Preview):** Stream the PDF from the API to a temporary cache. Show a "Draft" watermark if not finalized.
- **After Download:** Load from the local encrypted `filePath`. Enable offline viewing.

---

## 3. Outside the App (System Storage)

When a user opens the PDF from their phone's file manager:

### File Naming Convention
`[CASE_ID]_[DOC_TYPE]_[TIMESTAMP].pdf`
*Example:* `CASE-1001_MedicalReport_20231027.pdf`

### Metadata Properties
- **Title:** [Document Type] - [Child Name]
- **Author:** Adoption & Child Care System
- **Subject:** Official Case Documentation
- **Keywords:** Adoption, Child Care, [County Name]

### Security Features
- **Watermarks:** "OFFICIAL COPY" background text.
- **Encryption:** (Optional) Password-protected PDF using the User's ID or unique PIN.
- **Non-Editable:** The PDF should be exported as a flattened file to prevent tampering.

---

## 4. Technical Implementation Recommendations

### PDF Generation (Backend/Node.js)
Use `pdfmake` or `puppeteer` to render HTML templates into high-quality PDFs on the server side.

### PDF Viewing (Android)
- **Native:** Use `android.graphics.pdf.PdfRenderer` for simple rendering.
- **Library:** Use `com.github.barteksc:android-pdf-viewer:2.8.2` for a more polished UI with zoom and animations.
- **Intent:** Use `FileProvider` to open the PDF in the user's preferred external viewer (Adobe, Drive, etc.).
